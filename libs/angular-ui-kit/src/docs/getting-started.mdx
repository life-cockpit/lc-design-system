# Installation

## 1. Install the package

```bash
npm install @life-cockpit/angular-ui-kit
```

## 2. Include styles

In your `styles.scss` (or via `angular.json` under `styles`):

```scss
@use '@life-cockpit/angular-ui-kit/styles';
```

Or individually:

```scss
@use '@life-cockpit/angular-ui-kit/styles/reset';
@use '@life-cockpit/angular-ui-kit/styles/theme-light';
@use '@life-cockpit/angular-ui-kit/styles/theme-dark';
@use '@life-cockpit/angular-ui-kit/styles/typography';
@use '@life-cockpit/angular-ui-kit/styles/utilities';
```

## 3. Use components

All components are **standalone** and can be imported directly:

```typescript
import { Component } from '@angular/core';
import {
  ButtonComponent,
  InputComponent,
  CardComponent,
} from '@life-cockpit/angular-ui-kit';

@Component({
  selector: 'app-my-page',
  standalone: true,
  imports: [ButtonComponent, InputComponent, CardComponent],
  template: `
    <lc-card>
      <lc-input label="Name" placeholder="John Doe"></lc-input>
      <lc-button variant="primary" (clicked)="onSubmit()">
        Save
      </lc-button>
    </lc-card>
  `,
})
export class MyPageComponent {
  onSubmit() {
    console.log('Submitted!');
  }
}
```

## 4. Provide HttpClient (for Icons)

The `IconComponent` loads SVGs via HTTP. Make sure `provideHttpClient()` is included in your app config:

```typescript
// app.config.ts
import { provideHttpClient } from '@angular/common/http';

export const appConfig = {
  providers: [provideHttpClient()],
};
```

## 5. Theming

Components use CSS Custom Properties. Choose a theme by setting a CSS class on `:root` or `<body>`:

```html
<!-- Light Theme (default) -->
<html class="light">

<!-- Dark Theme -->
<html class="dark">
```

## 6. Component Theming (Header, Sidenav, Logo)

Some components support an explicit `theme` input, allowing them to have a dark or light appearance **independently** of the global page theme.

### Header

```html
<!-- Follows global theme (default) -->
<lc-header [theme]="'auto'" [title]="'My App'"></lc-header>

<!-- Always dark, regardless of page theme -->
<lc-header [theme]="'dark'" [title]="'My App'"></lc-header>

<!-- Always light -->
<lc-header [theme]="'light'" [title]="'My App'"></lc-header>
```

The header sets internal CSS tokens that you can override without touching global neutrals:

| Token | Purpose |
|---|---|
| `--lc-header-bg` | Background color |
| `--lc-header-fg` | Primary text/icon color |
| `--lc-header-fg-secondary` | Secondary text (subtitle, email) |
| `--lc-header-border` | Bottom border color |
| `--lc-header-hover-bg` | Hover background |
| `--lc-header-trigger-border` | Profile button border |
| `--lc-header-trigger-fg` | Profile button icon color |

### Sidenav

```html
<!-- Always dark sidenav with light page -->
<lc-sidenav [theme]="'dark'" [isOpenInput]="true" modeInput="docked"></lc-sidenav>
```

Internal tokens:

| Token | Purpose |
|---|---|
| `--lc-sidenav-bg` | Background color |
| `--lc-sidenav-fg` | Item text color |
| `--lc-sidenav-fg-active` | Active/hover text color |
| `--lc-sidenav-border` | Border color |
| `--lc-sidenav-hover-bg` | Item hover background |
| `--lc-sidenav-section-fg` | Section title color |

### Logo

```html
<!-- Inverts for dark backgrounds -->
<lc-logo [colorMode]="'dark'" variant="full" size="md"></lc-logo>

<!-- Follows global dark/light automatically -->
<lc-logo [colorMode]="'auto'" variant="full" size="md"></lc-logo>
```

The `colorMode="dark"` applies `filter: invert(1) hue-rotate(180deg)` to maintain detail while adapting to dark backgrounds.

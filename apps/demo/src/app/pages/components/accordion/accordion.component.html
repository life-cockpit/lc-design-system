Accordion Component

Accordions display collapsible content sections. They are useful for organizing large amounts of content into expandable panels, FAQs, settings groups, and filter sections.

Variants

Two visual variants: outlined (with border) and flat (no border).

Uses a border around the accordion. Best for clearly delineated sections.

No border. Works well when nested inside cards or other containers. 

<lc-accordion title="Outlined" variant="outlined">...</lc-accordion>
<lc-accordion title="Flat" variant="flat">...</lc-accordion>

Sizes

Three size options control header and body padding.

Compact padding — ideal for dense layouts or nested accordions.

Standard padding — works for most use cases.

Generous padding — best for primary content sections. 

<lc-accordion title="Small" size="sm">...</lc-accordion>
<lc-accordion title="Medium" size="md">...</lc-accordion>
<lc-accordion title="Large" size="lg">...</lc-accordion>

Expand & Collapse

Use two-way binding with [(expanded)] to control the open/close state. Click the headers below to toggle them.

Disabled State

Disabled accordions cannot be toggled.

You should not be able to see this content.

Content is visible but the accordion cannot be toggled. 

<lc-accordion title="Disabled" [disabled]="true">...</lc-accordion>

API Reference

Property Type Default Description
title string required The title text shown in the header
expanded boolean false Whether the panel is open — supports two-way binding
variant 'outlined' | 'flat' 'outlined' Visual style of the accordion
size 'sm' | 'md' | 'lg' 'md' Controls header and body padding
disabled boolean false Whether the accordion can be toggled
